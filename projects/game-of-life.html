JavaScript Game of Life

Project Overview

JavaScript Game of Life is a console-based implementation of Conway's Game of Life written in plain JavaScript. The project simulates generations of cells in a two-dimensional grid, where each cell is either alive or dead. The simulation runs automatically in the terminal and uses Unicode symbols and ANSI colors to make the output easier to read.

Problem Statement and Objectives

The goal of this project was to practice JavaScript logic, arrays, loops, functions, and simulation behavior by building a working version of Conway's Game of Life. The main objective was to apply the standard Game of Life rules, generate random starting patterns, count neighboring cells, and display each new generation clearly in the console.

Technical Implementation

The project uses a two-dimensional array to represent the grid. Each cell is stored as either 0 for dead or 1 for alive. A random starting grid is generated, and then the program repeatedly creates new generations using Conway's rules. The code separates responsibilities into helper functions for grid creation, random cell generation, neighbor counting, rule calculation, and console rendering.

The simulation uses setInterval to update the grid every second. ANSI escape codes are used to color the terminal output, with yellow squares representing live cells and blue dots representing dead cells.

Challenges and Solutions

One challenge was correctly counting the eight neighboring cells around each position without causing errors at the edges of the grid. I solved this by checking whether neighboring rows and columns exist before adding their values. Another challenge was making sure each generation was based on the previous grid instead of changing cells too early. To solve this, the program creates a new empty grid for each generation and stores the updated cell values there.

Results and Impact

The completed project demonstrates core JavaScript programming skills, including working with arrays, nested loops, conditional logic, helper functions, and timed execution. It also helped me better understand how small rules can create complex behavior over time in a simulation.

function makeNewGeneration(grid) {
    let result = makeEmptyGrid(grid.length, grid[0].length);
    for (let i = 0; i < grid.length; i++) {
        for (let j = 0; j < grid[i].length; j++) {
            const neighboursCount = getNeighboursCount(grid, i, j);
            result[i][j] = getNewCellValue(grid[i][j], neighboursCount);
        }
    }
    return result;
}

This function creates the next generation without modifying the original grid while it is still being checked. This is important because every cell must be updated based on the same previous generation.
