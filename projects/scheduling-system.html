Full-Stack Scheduling System

Project Overview

The Full-Stack Scheduling System is a web application built with Node.js, Express, HTML, CSS, and JavaScript. The application allows users to load a weekly schedule, select available time slots, and save schedule changes through client-server communication.

Problem Statement and Objectives

The goal of this project was to build a simple scheduling system that demonstrates both front-end and back-end development. The project needed to display a weekly schedule, allow users to interact with available and booked time slots, and send updated schedule data to the server.

Technical Implementation

The front end uses JavaScript to dynamically build the schedule table, create buttons for each time slot, and update the interface based on the current user. The back end uses Express routes to serve static files, receive schedule updates, return schedule data as JSON, and handle basic email-based role detection.

The schedule is stored as an array of objects, where each row represents a time slot and each weekday stores the current availability value. The client uses fetch requests to communicate with the server and update the displayed schedule after data is saved.

Challenges and Solutions

One challenge was keeping the client-side schedule and saved server data in sync. I solved this by storing both the current schedule and the last saved schedule, then comparing them to detect unsaved changes. Another challenge was displaying different button behavior for instructors and students. I solved this by using email-based role detection and separate logic for available, booked, disabled, and student-booked schedule cells.

Results and Impact

This project helped me practice full-stack development with JavaScript, including DOM manipulation, event handling, JSON data, fetch requests, Express routes, and basic server-side logic. It also gave me experience designing an interactive interface that changes based on user role and saved application state.

Future Enhancements

  • Store schedule data in a database instead of server memory
  • Add real user authentication instead of email-based role detection
  • Add multi-user support so several people can use and update the schedule
  • Add real-time updates so users can see schedule changes without refreshing the page
  • Add conflict handling to prevent two users from booking the same time slot at the same time
  • Improve the user interface for mobile and smaller screens
